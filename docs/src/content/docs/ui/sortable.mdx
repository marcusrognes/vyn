`@vynjs/ui/sortable` turns a list into a reorderable one. Items can
be dragged with the mouse, picked up with the keyboard, and dropped
into a new position. Emits a `reorder` event with the moved item and
its new index; the consumer updates the data.

Add a matching `data-sortable-group="<name>"` to multiple containers
and items move between them — same positional drop, same `reorder`
event per container, plus a `move` event on the destination when an
item crosses lists. No need to bolt [`drag-drop`](/vyn/ui/drag-drop/)
on top; reach for that when you also need typed drops, multi-drag, or
trash-style targets.

```ts
import "@vynjs/ui/sortable";

document.querySelector("ul")!.addEventListener("reorder", (e) => {
	const { order } = (e as CustomEvent<{
		order: { index: number; id: string }[];
	}>).detail;
	// persist the new order
});
```

## Attributes (on the container)

| Attribute | Type | Default | What it does |
|---|---|---|---|
| `data-sortable`       | boolean | required | Activates the behavior |
| `data-sortable-group` | string  | unset    | Containers sharing the same group accept items dragged from each other |

## Attributes (on items)

| Attribute | What it does |
|---|---|
| `data-id` | Stable id surfaced in the `reorder` and `move` events. Falls back to `id` if absent |

## Events

| Event | Fires on | Detail | When |
|---|---|---|---|
| `reorder` | the container | `{ order: { index, id }[] }` | After drop or keyboard commit, with the container's new order |
| `move`    | the destination | `{ id, from, to }` (both `HTMLElement`) | After a cross-container drop completes. Fires **after** the two `reorder` events for source and destination |

A within-container reorder fires `reorder` once. A cross-container
drag fires `reorder` on both the source and the destination, then
`move` on the destination.

## Keyboard

When an item (or its handle) has focus:

| Key | Effect |
|---|---|
| `Space`, `Enter` | Pick up / drop the item |
| `ArrowDown`, `ArrowUp` (after pickup) | Move down/up one position |
| `Home`, `End` (after pickup) | Move to first / last |
| `Esc` (after pickup) | Cancel; restore original position |

While picked up, the item visually lifts (via `data-state="picked"`)
and a live region announces "Item picked up. Press up and down to
move, Enter to drop, Escape to cancel."

## ARIA and `data-state`

| Attribute | When |
|---|---|
| `aria-grabbed="true"` | On the picked-up item |
| `data-state="picked"` | Same; for CSS targeting |
| `data-state="dragging"` | On the item being dragged with a pointer |

## Touch and pointer

The behavior uses Pointer Events, so touch, stylus, and mouse all
work. On touch, a 300ms long-press picks up the item (avoiding
conflict with scroll); set `data-handle` to a dedicated handle to
remove the delay.
